create a project using npm init
package.json file is created
create src folder where we can create node js pages and main js file which has all other js code.
install expressjs - npm i express
Node modules folder is created which has  all the dependencies available to run express application.
package-lock.json is created which has all the current dependencies with current versions.
"express": "^5.2.1"  - 
5 is major version(any change that break the existing feature can be known as major, can break application), 
2 is the minor(any new features pushed in future, and backward compatible) version and 
1 is the patch(any change like bug fix that can be known as patch).
caret ^ - automatically updated if any minor or patch change.
tilde ~ - automatically updated if any patch change.
install nodemon that will restart server when any changes are made.
to create a server
1. import express using the commonjs or es modules
    const express = require("express");
2. create an app that listen to the project
    const app = express();
    app.listen(3000, ()=>{ 
        console.log("App is running")
    })//will take two parameters one is port number and other is callback function to show app status.
3. .use is used to create requests and response for the server.
    app.use((req, res)=>{
        res.send("The app is running on server")
    })
4. Update script with "dev" nodemon src/app.js

In routing the order of defining routes is important. because the path is checked and matched line by line.
ex: app.use("/hello", (req, res) => {//the callback is known as route handler
    res.send("hello");
    });
    app.use("/hello/2", (req, res) => {
    res.send("hello 2");
    });
    this will always hello when any of above path is called.
now,
    app.use("/hello/2", (req, res) => {
    res.send("hello 2");
    });
    app.use("/hello", (req, res) => {
    res.send("hello");
    });
    this will print hello for /hello and hello2 for /hello/2
use .get or .post to match exact API call

HTTP methods - get, post, patch, put, delete
get - when we run an api in browser it will by default makes GET api call.
post - we have to write explicit code to make POST api call.

To test the API, we should always use postman.

if, 
app.delete("/ab?c", (req, res) => {
  res.send("Data deleted from data base");
});
the API works with the end point if it is abc or ac.

if, 
app.delete("/ab+c", (req, res) => {
  res.send("Data deleted from data base");
});
the API works with the end point if it is abc or abbbbbbbc or abbc.
works when API endpoint starts with ab and ends with c

if, 
app.delete("/ab*cd", (req, res) => {
  res.send("Data deleted from data base");
});
the API works with the end point if it is abcd or abasfasfasfsdfksdfgsdfkdasgcd.
works when API endpoint starts with ab and ends with cd


if, 
app.delete("/a(bc)?d", (req, res) => {
  res.send("Data deleted from data base");
});
In this API the bc is optional.
We can also use regex for path.

use the req.query to read query parameters

use the user/:userId to pass anything after API endpoint
use req.param to get parameter
: means dynamic routes

Make sure to send response from the server for sure. if not the call fails after being into infinite response checking loop

Routing and Middle ware
If the API has more than one routing handler we will get response for only first response. if there is no response from first handler the API will be in infinite loop and dont go to next handler.
to overcome this we need to use next() function to see the next response. 
if first handler sends response the next handler code is read and throws error because the TCP connectino was already made and response sent by first handler.
next() alwasy calls the next function.
if there is no reponse send with in multiple route handlers and next() is used then we see the API repsonse as cannot find user. if no next in final route handler then it will be infinite loop.
we can also add multiple routes with array[]    

Middle ware - 
The functions used in the HTTP call is called middle ware.
In an api call there can be number of middlewares but the request handler that sends the response will be executed and response send.

Diff b/w app.use and app.all

In general middle ware can be used with app.use() to check authorization before any api is called related to the particular requests.
